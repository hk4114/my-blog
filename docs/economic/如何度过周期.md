# 在“内卷”与周期中，我该如何寻找新机会

## 如何度过周期
一、保持健康
减肥不是有始有终的“项目”，而是从不健康的生活方式到健康的生活方式的过程。
生活习惯不改变，减多少次肥都没用。培养健康的生活方式，比如到点睡觉，少吃多动。
体重下降和瘦，只是这个生活方式的“副作用”。

二、保持记录
人生是不断收集回忆的过程，最终能够陪伴我们的也只有回忆。
发现以前自我厌恶感太强，对自身状态记录得太少，手机里只留下了谁都能拍的风景，却没有留下风景里的自己。
所以从 23 年开始，我开始每天写日记，睡觉前用 5 分钟复盘当天的收获与教训。
毕竟生前不纪念自己的人，死后就会没人纪念。

三、保持节俭
存下 100 万并非真正的难点，真正的挑战是成为那个能存下 100 万的人。人们往往不是被市场打败，而是被自身的欲望击溃。能否存下 100 万，关键不在运气，而在于能否忍受无聊。
致富并非源于聪明，而是源于在无聊时不去浪费金钱。
不克制欲望，年薪百万的“高薪囚徒”比低收入者更危险——他们只是用更贵的方式穷着。
给自己定下100万的财产目标，是因为第一个 100 万是积累过程中最缓慢、最痛苦且最折磨的阶段。
在此期间，回报极其微小，资金增长缓慢，仿佛在嘲弄努力。
本质还是克制自己的欲望。利用欲望让自己努力，控制欲望让自己的生活轻盈。
数字技术让我们获取信息变得异常便利，让我们更容易提升自己的能力，同时也更容易产生欲望。但能力提升的速度是多项式级别的，而欲望则很容易以指数级速度增长。想减轻焦虑，提升能力是可选项，控制欲望是必选项。
在工作上，要向积极性最高的同志看齐；在生活上，要向水平最低的同志看齐。——雷锋

## 赚钱的底层逻辑：万变不离其宗的“套利”

商业社会看似充满各异的机会、赛道、风口和竞争态势，从国库券、股市、直播到台球厅，具体形态千差万别。
但是本质从未改变。归纳起来，所有的赚钱方式都离不开三个要素：价差、利差、汇差。而实现这些的核心，在于信息差。

市场刚出现时信息差充足，套利机会多；成熟后信息差被抹平，竞争加剧。如同热力学第二定律，导致系统熵增，“机会”从非平衡态走向平衡态（内卷）。
无论是早期国库券、外汇、原始股，还是新兴产业，一旦进入“卷”的阶段，就意味着机会在减弱，需要转向差异化。
关键是让自己成为信息差，形成护城河，否则难以避免内卷。财经 IP、财经 MCN 之所以风险高，是因为复制成本低、缺乏护城河。

## 宏观周期的剧变：从“资产端”到“负债端”
过去 20—30 年，中国更多关注资产端回报，即资产升值，例如房价从 100 涨到 200、300、500。负债端回报（如租金现金流）被长期忽视。  
如今环境变化，资产端回报下降，高股息红利成为主流逻辑：现在的核心不再是“资产增值”，而是“负债端回报”，也就是当下的现金流。
无论是一级市场的投资人，还是二级市场的股民，现在的判断标准都变成了：你有利润吗？你有分红吗？你有现金流吗？ 单纯靠烧钱、讲故事、博高估值的模式已经行不通了。
对于个人而言，这意味着职业规划和投资都要追求“稳”。为什么现在年轻人热衷于考公、考编？这和当年日本经济泡沫破裂后，年轻人追求终身雇佣制是一样的逻辑——大家都在寻求稳定的负债端收益（工资），以抵御资产端的波动

## 商业模式的迭代：传统行业的重新爆发
> 从“商品附属”到“体验核心”

在一个行业从0到1的爆发期，凭借信息差去“挖金子”；当行业进入1到10的红海期，竞争激烈时，要学会转身去“卖铲子”（提供供应链、服务或方法论）。
商业模式的迭代：传统行业的重新爆发
虽然宏观环境变了，但并不意味着没有机会。只要愿意深入观察，传统行业通过商业模式的迭代，依然能焕发新生。
* 台球厅与网吧的进化： 现在的台球厅不再是单纯打球的地方，它融合了商务KTV、酒吧和社交属性，成为了年轻人的“平替”娱乐场所。网吧也不再是脏乱差的小作坊，而是进化成了提供高性能电脑的“高端咖啡厅”，通过高毛利的餐饮来盈利，满足了自由职业者和电竞爱好者的场景需求。
* 预制菜与反预制菜的博弈： 外卖行业为了效率大量使用预制菜，导致同质化竞争。但反过来，那些打出“现炒”、“非预制”、“需等待”招牌的餐厅，虽然价格贵、时间长，却因为满足了追求健康的细分人群而异军突起。
* 实体女装的社交化： 电商冲击下，高端实体女装店依然存活，秘诀在于将店铺变成了“社交会所”。通过举办走秀、下午茶，维护核心客户群的情感链接，卖衣服成了社交的附属品。

这说明，即使是传统行业，只要能理解用户需求的迭代，从简单的售卖商品转向提供情绪价值、社交场景或极致品质，依然有巨大的生存空间。

## 给年轻人的建议：构建“兴趣-专业-商业”的闭环
1. 个人需要对未来 5—10 年做预判，宏观有用，微观调研也有用。真正的框架是从宏观趋势到产业中观，再到个人微观逐层思考。  
2. 选择的重要性被强调：99% 是选择，1% 才是努力。选择错了，汗水就不值钱。年轻人应多观察、多交流，找行业从业者聊，获取信息差。商业底层逻辑万变不离其宗。
3. 主业可不放弃，但关联领域要延展。比如把金融日记从彭博笔记搬到公众号，时间成本不高，却打开了传媒领域的通道。 

年轻人可以按兴趣找工作，但不能只凭兴趣。兴趣要转成专业，再加入商业思考，形成闭环。 
以电竞为例，不仅要喜欢比赛，还要研究赛事、直播、战队运营与成本结构。兴趣必须先能养活自己，再谈热爱与发展。  
很多年轻人专业兴趣深，但因无法商业化而难以生存，播客行业就是典型现象。
要培养思维方式，看到反直觉现象就追问原因。获取信息差的关键在于多问、多聊、多研究。

- 兴趣—专业—商业三环要形成闭环，这是最可持续的路径。  
- 不能再假设高增长环境，要在宏观变化中调整预期，先稳住现金流。  
- 保持敏锐度与观察力，不要单纯躺平，运气不差也能过得不差。

### 2. 建立“三环闭环”
很多年轻人有兴趣（比如爱打游戏），甚至有专业度（游戏打得好），但依然养不活自己，因为缺了最关键的一环——商业思考。
真正的成功是把兴趣做成事业。这需要你构建一个闭环：
* 第一环：兴趣。 这是一个人的原动力。
* 第二环：专业深度。 将兴趣打磨成超越常人的技能。
* 第三环：商业模式。 思考这个东西怎么变现？产业链上下游是谁？我在其中能提供什么独特价值？
只有打通了这三环，才能从一个“爱好者”变成一个“精通者”和“受益者”。

### 3. 保持“挖金子”与“卖铲子”的思维切换
在一个行业从0到1的爆发期，凭借信息差去“挖金子”；当行业进入1到10的红海期，竞争激烈时，要学会转身去“卖铲子”（提供供应链、服务或方法论）。

### 4. 保持好奇与跨界
不要被自己的专业或标签束缚。做金融的可以做自媒体，送外卖的可以研究供应链。在这个技术快速迭代的时代（如AI的出现），**乱拳打死老师傅**是常态。保持对新事物的探索欲，不拒绝任何形式的延展，是应对不确定性最好的武器。

## 总结
未来五到十年，或许不再有遍地黄金的运气，但对于那些能够看清宏观大势、拥有专业护城河、并且时刻保持商业敏感度的年轻人来说，机会依然存在。与其抱怨环境，不如先让自己成为那个拥有“信息差”的人。

CLAUDE.md 是放在项目根目录的文件，Claude Code 每次启动都会读取它。你可以在里面写代码规范、设计原则、PR 模板、常见错误提醒——任何你希望 Claude 记住的东西。

具体 prompt 可以是：“Update your CLAUDE.md so you don't make that mistake again.”

团队里有个工程师的做法更系统：他为每个项目/任务维护一个 notes 目录，每次 PR 后更新。然后在 CLAUDE.md 里指向这些 notes，相当于给 Claude 建了一个持续更新的知识库。

这个思路跟我经常提到的 Skills 迭代思路类似。通常一开始规则不够完善，每次遇到问题，基于当前上下文让 Agent 自己去完善是效果最好的。一方面你不需要从头描述问题，另一方面 Agent 很善于归纳总结。

至于维护一个 notes 目录，是个蛮好的积累经验的实践。这事你不需要自己写，可以做一个 Skill，每次让 Claude Code 从当前会话中复盘，提炼成 Note。甚至可以连上 hook，每次会话结束自动执行。不过这条我个人不推荐，太多没意义的信息不见得是好事。

这个技巧的核心是把人脑里的经验变成系统知识。时间越长，CLAUDE.md 越完善，Claude 犯的错越少，你需要纠正的次数也越少。这是一种复利。

这里需要补充一个注意事项，CLAUDE MD 不建议放太多内容，只会适得其反，只放最重要的 AI 没训练过的内容，更多的内容作为文件链接按需读取。

很多人把设计模式、规范、最佳实践之类的都放进去，先不说这些 AI 都训练过，你最多说个名字就够了，就算是你需要的，也不是每次都要，不如放一个链接或者移到 Skills 按需加载。

## 提升你的 Prompting 技巧
第一招：让 Claude 来考你
Prompt 示例：“Grill me on these changes and don't make a PR until I pass your test.”（针对这些改动考我，直到我通过测试才能提 PR。）

或者：“Prove to me this works.”（向我证明这个能 work。）让 Claude 对比 main 分支和你的 feature 分支的行为差异。

这相当于把 Claude 从“执行者”变成了“审核者”。让它反过来 review 你。

第二招：推倒重来
当 Claude 给出的方案不够好，不要在上面打补丁。直接说：“Knowing everything you know now, scrap this and implement the elegant solution.”（基于你现在知道的所有信息，扔掉这个方案，实现一个更优雅的版本。）

我通常会用 git 把代码回滚到修改前，然后新开会话、调整提示词重来。在当前会话继续的话，之前的错误信息可能会干扰 Claude 的判断。

第三招：减少歧义
交代任务时，spec 写得越详细越好。你越具体，Claude 的输出越准确。这听起来像废话，但很多人（包括我）还是习惯性地写模糊的需求，然后抱怨 AI 不懂。

其实人都懒，能一句话说清楚肯定懒得说第二句。用 Plan 模式相对好一点，你能知道它听懂了没有。

## 终端和环境配置

## 使用 Subagents
这是一个进阶技巧，用好了很强大。

最简单的用法： 在任何请求后面加上“use subagents”。Claude 会自动把任务拆分给多个 Subagents 并行处理，相当于让它“开更多的线程”来解决问题。

另一个用法是用 Subagents 保持主会话的上下文干净。把一些独立的子任务分派出去，主会话只负责整体协调。这样主会话的 context window 不会被塞满中间过程。

Subagents 可以让任务并行，大大节约时间。比如我之前给文章生成插图的时候，就会让它跑 4 个 Subagents，把提示词文件路径传给每个 Subagent。不过实际用下来稳定性还不够，经常会有 Subagent 挂掉的情况，期待后续版本改进。

更高级的玩法：用 hook 把权限请求路由给 Opus 4.5（Anthropic 最强的模型），让它判断哪些操作是安全的可以自动批准，哪些需要人工确认。相当于给 Claude 加了一个“安全审核员”。

参考文档：https://code.claude.com/docs/en/hooks#permissionrequest

## 用 Claude Code 做数据分析
这个用法可能出乎很多人意料。

Anthropic 团队把 BigQuery 的使用封装成了一个 skill，所有人都可以在 Claude Code 里直接用 bq 命令行查询数据。Boris 说他已经六个月没写过一行 SQL 了。

这不限于 BigQuery。任何有 CLI、MCP 或 API 的数据库都可以这样用。PostgreSQL、MySQL、MongoDB，都可以让 Claude 帮你写查询、跑分析、生成报告。

对于非工程师来说这可能更有价值。团队里的数据科学家们现在也在用 Claude Code 写查询、做可视化。工具的边界正在模糊。

## 用 Claude Code 学习
最后这个技巧是关于怎么用 Claude Code 来学习新东西。

首先，在 /config 里开启“Explanatory”或“Learning”输出风格。这样 Claude 在改代码的时候会解释“为什么”这么改，而不只是改完拉倒。

第二个用法：让 Claude 生成 HTML 幻灯片来解释不熟悉的代码。Boris 说效果出奇的好。你可以直接在浏览器里看一个图文并茂的代码讲解。

第三个用法：让 Claude 画 ASCII 图来解释协议、架构、数据流。纯文本的图表意外地有助于理解复杂系统。

## 三层防御 + 一击必杀
第一层：认知过滤（不参与愚蠢的游戏）
- 预期 > 价格：市场炒的是故事，不是数字。别在 price action 里找真相，在 narrative 崩溃前离场。
- 事实 > 希望：希望是成本，事实是利润。踏实的本质是去除幻觉后的重量感。
- 认知 = 持仓上限：拿不住的钱不属于你，那是认知税。逻辑不闭环，仓位就是定时炸弹。

第二层：行为纪律（控制下行）
- 不懂就停：这是能力圈的铁门。贸然行动不是勇敢，是缴械投降。
- 减少错误 > 追求正确：投资是负和游戏，先不输。芒格说的"反过来想"就是这意思。
- 三忌（跟随、随便、规模）：跟随是羊群，随便是懒惰，规模是傲慢。独立、审慎、聚焦，缺一不可。

第三层：时机与杠杆（上限放大）
- 确定性下重注，不确定用杠杆：凯利公式的肉身实践。概率高时 all-in，概率模糊时用小成本试探（期权/轻仓）。
- 杠杆看成本：资金成本、时间成本、机会成本。算账不清楚的杠杆是自杀。
- 多方案比较：必须有 Plan B，而且 Plan B 的缺陷你必须能吞下。

#### 1. 关于“认知过滤”

在你的表格中，**国电南瑞**和**紫金矿业**是认知门槛最高、但逻辑最稳固的。南瑞的90%市占率不是炒出来的，是技术代差和国家意志的体现。这符合你说的“踏实的本质是去除幻觉后的重量感”。

#### 2. 关于“第二层思维”

当市场都在炒作光伏组件价格战时，第二层思维会看向**特变电工**和**东方电气**。光伏装得越多，电网的消纳压力越大，对变压器、调峰燃机的需求就越是“刚需”。这才是真正“反过来想”的结构性机会。

#### 3. 关于“集中力量”与“信念”

你提到的“5个50%不如1个250%”，这在**紫金矿业**上体现得最明显。如果你的认知能覆盖全球铜金矿的供需结构，那么在周期波动中，紫金就是那个可以让你“拿得住”的仓位。

NotebookLM真正的能力在于跨文档的模式识别。它能同时在工作记忆中保持50多个来源，并即时交叉引用。这是人类大脑做不到的事情。如果喂给它正确的指令，它可以把10小时的分析压缩到20秒完成。

以下是经过验证真正有效的十个提示词：

一、专家综合器

让AI扮演某领域15年经验的专家，从资料中识别出3个从业者会立刻认为具有突破性的核心洞见，并解释每个洞见为何重要、挑战了哪些传统认知。这个提示词强制深度优先于广度，输出可以直接使用。

二、矛盾猎手

比较多个来源，找出所有相互矛盾的观点。对每个矛盾点，判断哪个来源的证据更强，如果两者都可信，解释可能导致分歧的因素。做文献综述和尽职调查时，这能省下大量手动交叉核对的时间。

三、实施蓝图

提取所有来源中提到的可执行步骤、工具、框架和技术，整理成包含前置条件、预期结果和潜在陷阱的分步实施计划。大多数研究者从不做这种从理论到实践的转化工作，而这恰恰是知识产生价值的关键环节。

四、问题生成器

基于现有资料，生成15个专家会问但这些资料没有回答的问题，优先考虑能推动领域发展或揭示当前理解关键缺口的问题。这是发现原创研究空白或产品机会的方法。

五、假设挖掘器

识别资料中所有未明说的假设，对每个假设评估其关键程度和出错可能性，解释如果该假设为假会改变什么。大多数突破性洞见来自质疑那些没人注意到的假设。

六、框架构建器

创建一个整合所有来源概念的综合框架，包括关键组件、组件间关系、应用决策树，以及框架失效的边缘情况。框架是将知识操作化的方式，这个提示词能自动构建它们。

七、证据地图

为每个主要论断提取支持证据，并评估其强度：轶事级、相关性、实验性还是元分析。标记那些证据薄弱却表述得很有把握的论断。这是几秒钟内达到发表级别严谨度的方法。

八、受众翻译器

将洞见翻译给三类不同受众：高管、工程师、终端用户。针对每类受众关心的内容，使用他们能立即理解的语言和案例。一次上传，三种可用输出。

九、时间线构建器

提取所有日期、事件、里程碑和时间引用，构建展示领域演变的完整时间线，识别进展显著加速的拐点。这对理解发展势头和预测未来走向非常有用。

十、弱点探测器

扮演严苛的同行评审，识别每个方法论缺陷、逻辑漏洞、过度声称和无支撑的跳跃，并建议需要什么额外证据来加强论证。这能帮你避免在摇摇欲坠的地基上建造大厦。

这些提示词适用于学术研究、市场分析、尽职调查、竞争情报、产品研究和行业深度调研。

工具的强大程度取决于使用者的指令质量。NotebookLM的问题从来不是能力不足，而是指令不足。
