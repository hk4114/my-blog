## AI 辅助编程
[writing-a-good-claude-md](https://www.humanlayer.dev/blog/writing-a-good-claude-md)

[compounding-engineering-plugin](https://github.com/EveryInc/compounding-engineering-plugin)

在使用 AI 辅助编程时，最难的往往不是写代码，而是如何让 AI 严格遵守团队已有的工程规范，而不是每次都“随心所欲”地生成代码。

偶然发现 Compounding Engineering Plugin 这个开源项目，它提出了一套非常硬核的 Plan-Work-Review 工程化工作流，旨在让 AI 的每一次开发都能积累质量，而非制造技术债务。

它不是简单的“记忆存储”，而是通过在 Plan 阶段深度分析代码库模式生成详细 Issue，再利用隔离环境（Worktree）执行开发，最后由 10 多个专用 Agent 并行审查。

最有意思的是它的 Review 机制，内置了如 `security-sentinel`（安全哨兵）和 `performance-oracle`（性能预言家）等专业 Agent，像人类高级工程师一样对代码进行全方位体检。

[ccNexus](http://github.com/lich0821/ccNexus)

如果你希望 Claude Code 不仅仅是一个“代码生成器”，而是一个能遵守纪律、不仅能写还能自我审查的“虚拟同事”，这个插件非常值得研究。

使用 Claude Code，有时会碰到 API 调用不稳或者达到用量限制，导致代码生成中断，然后手动切换 API Key 再重启 CC，感觉相当麻烦。

因此有位开发者，在 GitHub 上开源了一个 Claude Code 智能 API 轮换代理工具：ccNexus。

提供了直观的可视化管理界面，支持添加 Claude、OpenAI、Gemini 等多种 API 格式。

当在 Claude Code 遇到某个 API 调用失败了，可以自动轮换到下一个 API，无需重启 CC。

同时还可以实时统计我们使用的情况，包括请求、错误和 Token 使用，在界面上直观显示。

[Claude Code Viewer](https://github.com/d-kimuson/claude-code-viewer)
目前主流的终端 AI 编程工具 Claude Code，虽然功能强大，但是命令行操作，始终阻碍着部分技术小白上手使用。
可以看一下， 这个开源工具，它为 Claude Code 提供了一个相对功能完整的直观 Web 操作界面。
不仅支持实时查看对话日志、启动和恢复会话，轻松管理所有对话，还内置了 Git 差异对比和提交功能，方便审查和提交代码。
除此之外，还支持上传图片、PDF 和文本文件，每种格式都有专门的预览组件，以及可以设置定时任务，在达到限制恢复后自动继续执行任务。
通过 npm 全局安装或直接运行即可使用，支持 macOS 和 Linux 系统，想要使用 Claude Code 的初学者可以看下。

[superpowers](https://github.com/obra/superpowers)

用 Claude Code 写代码，每次都要从头解释需求、工作流程，重复的调试步骤说了一遍又一遍,给人感觉用 AI 编程的效率反而不高。

无独有偶，在 GitHub 上找到 Superpowers 这个技能库，专门给 Claude Code 提供系统化的工作流程和最佳实践。

包含测试、调试、协作、开发等几大类技能。写代码时自动按测试驱动开发（TDD）流程，调试问题走系统化四步法，提交前自动验证。还有头脑风暴、制定计划、批量执行等斜杠命令。

通过 marketplace 安装插件就能用，技能会根据任务自动激活。比如写功能时先写测试再写代码,调试时自动追踪根本原因。

如果你用 Claude Code 开发，想建立一套系统化的工作流程而不是随机应变，这个技能库值得试试。

[how-to-build-a-coding-agent](http://github.com/ghuntley/how-to-build-a-coding-agent)

想要了解 Cursor、Cline 这些 AI 编程工具内部的编码 Agent，发现网上资料很少，而且原理和实现细节没有提及。

恰巧，在 GitHub 上看到了 how-to-build-a-coding-agent 这个开源教程项目，手把手教我们从零构建一个功能完整的 AI 编程助手。

通过 6 个循序渐进的实战示例，从最基础的聊天机器人开始，逐步添加文件读取、目录浏览、命令执行、代码编辑和代码搜索等功能。

每个示例代码都开源，并包含详细实现步骤说明，同时配有架构图和流程图，帮助我们理解 Agent 的工作原理。

“人工智能正在发生变化。多年来，焦点一直放在那些擅长被动、离散任务的模型上：回答问题、翻译文本或从提示生成图像。尽管这种范式非常强大，但它需要在每个步骤中都由人类进行指导。现在，我们正在见证一种范式的转变，从仅仅进行预测或生成内容的人工智能，过渡到一种能够自主解决问题和执行任务的新型软件。

这一新前沿是围绕人工智能智能体（AI agents）构建的。智能体不仅仅是一个静态工作流中的 AI 模型；它是一个完整的应用程序，能够制定计划并采取行动以实现目标。它将语言模型推理的能力与实际执行的能力相结合，使其能够处理那些单独的模型无法完成的复杂多步骤任务。关键能力在于，智能体能够自主工作，找到实现目标所需的下一步，而不需要在每一个环节中都由人来引导。”

---- Prompt Start ----

Please generate a single-file React application for a "Retro Camera Web App" with the following specifications:

1. Visual Layout & Container Strategy

- Theme: Retro aesthetic with a "Handwritten" font style for all text.
- Title: "Bao Retro Camera" displayed at the top center.
- Instructions: Display usage instructions at the bottom right.
- Main Camera Container:
  - Create a fixed wrapper `div` that acts as the parent for the camera image, viewfinder, shutter button, and photo ejection slot.
  - Positioning: This container must be fixed at exactly 64px from the bottom and 64px from the left of the viewport (`bottom: 64px; left: 64px;`).
  - Dimensions: Width 450px, Height 450px.
  - Z-index: 20
  - All subsequent positioning coordinates (percentages) for camera elements are relative to this container.
- Background Image within Container:
  - Image Source: `s.baoyu.io/images/retro-camera.webp`
  - Size: 100% width and height of the container.
  - Position: Left 0, Bottom 0

1. Camera Functionality (The Viewfinder)

- Access the user's webcam.
- Viewfinder Position: The live video feed must be masked to a circle and positioned exactly over the camera lens.
- CSS for Video (Relative to Container): `bottom: 32%; left: 62%; transform: translateX(-50%); width: 27%; height: 27%; border-radius: 50%;z-index: 30`.
- Layering: The video must sit _above_ the camera base image visually but within the container.

1. Shutter & Photo Interaction

- Shutter Button: Create an invisible clickable area over the camera's button.
- CSS for Button (Relative to Container): `bottom: 40%; left: 18%; width: 11%; height: 11%; cursor: pointer;z-index: 30`.
- Action: When clicked, play a shutter sound effect and trigger the "Photo Ejection" animation.

1. Photo Ejection & Development Animation

- Aspect Ratio: The generated Polaroid-style photo card must strictly follow a 3:4 portrait aspect ratio (Vertical).
- Ejection Animation: The photo paper slides UPWARDS (negative Y) from behind the camera body.
- Layering: The photo must appear to emerge from _inside_ the camera (start with z-index(10) lower than camera body, then animate out).
- Ejection Container Origin CSS: `transform: translateX(-50%); top: 0; left: 50%; width: 35%;height: 100%;` (start position relative to the camera container).
- Ejection Container anmiation position from: ` translateY(0)` to ` translateY(-40%)`
- Developing Effect: Once the photo is taken, the image on the paper should transition from white/blurry to clear/sharp over a few seconds.

1. Drag & Drop "Photo Wall"

- Interaction: The user must be able to drag the ejected photo _out_ of the camera slot and drop it anywhere on the rest of the screen (the "Photo Wall").
- Drag Handle: The entire Polaroid card (the white frame and the photo) must be interactive. The user should be able to click and drag from any part of the card to move it.
- Logic: While developing, the photo is attached to the camera container. Once dragged, it becomes absolutely positioned on the main screen body.
- Freedom: Once on the wall, photos can be dragged and repositioned freely.

1. AI Integration & Text Interactions

- Caption Generation: Use the Gemini Flash API to analyze the captured image content.
- Prompt: Generate a warm, short blessing or nice comment based on the photo content.
- Language Requirement: The generated text language must match the user's browser language.
- Footer Layout: The bottom of the Polaroid paper (below the image) should display the current date and the AI-generated text.
- Text Interaction & Icons:
  - When hovering specifically over the text area, display two small icons:
    1. Pencil Icon: Enters edit mode.
    2. Refresh Icon: Re-triggers the AI generation for that specific photo to get a new caption.
- Editing Logic:
  - Trigger: Edit mode can be entered by clicking the Pencil icon OR by double-clicking the text itself.
  - Behavior: When editing, replace the text display with an input/textarea showing the raw text.
  - Controls: Pressing Enter saves the changes. Pressing Esc cancels the edit and reverts to the previous text.

1. Photo Controls (Card Level)

- Hover Actions
  : When hovering over a developed photo card on the wall (general hover), show a small toolbar at the top of the photo with: - Download Button: When clicked, this must render the entire Polaroid card (including the white frame, the photo, the date, and the handwritten caption) into a single image file and download it. (Recommended: use `html2canvas` or similar logic). - Delete Button: Removes the photo from the screen.

Technical Stack

- React (functional components).
- Tailwind CSS for styling.
- Framer Motion (recommended for complex dragging and layout animations).
- Lucide-react for icons.
- html2canvas (or similar) for snapshotting the DOM elements.
