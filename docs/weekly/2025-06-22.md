### mcp-prompt-server
平时用 AI 工具积累了一堆 Prompt，但真要用的时候总是想不起来放哪了，每次还得复制粘贴，颇为麻烦。

此时，可以尝试使用 MCP Prompt Server 这款 MCP 服务器，把我们常用的 Prompt 变成了可调用的工具，告别复制粘贴。

它将每个 Prompt 模板注册为 MCP 工具，支持自然语言调用，还能在 Raycast、Cursor、Windsurf 等工具中通用。

GitHub：github.com/joeseesun/mcp-prompt-server

主要功能：

- 丰富的内置 Prompt 模板，涵盖代码、写作、产品设计等场景；
- 所有 Prompt 自动注册为工具，支持参数化调用；
- 支持热加载，无需重启即可添加新 Prompt；
- 只需添加 YAML 文件即可扩展新功能；
- 适配 Raycast、Cursor、Windsurf 等主流编辑器；
- 自然语言对话即可调用，还能组合多个工具实现复杂工作流。

通过 npm install 安装后配置到 MCP 客户端即可使用。

### tchMaterial-parser
在 GitHub 上发现一款极方便下载中小学电子课本的开源免费工具：tchMaterial-parser。

它能帮助我们从智慧教育平台一键下载电子课本 PDF 文件，支持多本教材同时下载，自动用教材名称命名文件，还能显示实时下载进度，操作非常简单。

GitHub：github.com/happycola233/tchMaterial-parser

主要特性：

- 持 Access Token 登录，轻松获取受保护资源；
- 支持批量下载多个电子课本 PDF 文件；
- 自动使用教材名称作为文件名，方便管理；
- 优化 UI 以适配高分辨率屏幕，避免界面模糊；
- 实时显示下载进度，支持暂停/恢复操作；
- 支持 Windows、Linux、macOS 等多种操作系统。

提供 Windows 和 Linux 系统安装包，安装完设置 Access Token 即可使用，在 README 文件有详细说明。

公积金年报的微妙变化
公积金制度的存在实际上是一种制度性劫贫济富——少数人用多数人的钱购置资产。

居民不贷款vs“最后通牒博弈”
因为动物精神分为两类，乱纪元中求生存（避险），恒纪元中求发展（加杠杆）。
在行为经济学中，有一个“最后通牒博弈”理论——即A、B两个参与者，A是提议者、B是接受者。如果有100元要分配，A可以提议各自分配一半，也可以自己分配多一点，然后B来决定接受不接受。
理论上作为一个理性经济人，不管A给B分配多少，哪怕只分配1元，对于B来说也是完全新增的，所以看起来B应该怎么样都答应。
但实验结果显示，B对A所提出的不公平的分配会予以否决。比如A提议分配给B20%以下，B就会否决。尽管这样B会损失掉“净增20%的利益”，他们也要“阻止”A能轻松获得“80%利益”的不公平行为。
即心理学上的消极互惠。
我们常常在新闻中看到有人拿着id实名举报，尽管这样他（她）本人承担着很多风险和成本，他们也想要让“犯错”的人接受相应惩罚。
话说回来，前面谈到这新增的100元，在真实经济运行中，也并非完全“凭空”创造的，它借债务的名义，还是让B承担了经济成本的。所以B更加不会愿意A在分配上过于偏袒自己。
一旦B发现，就会产生“消极互惠”的念头。
这也就是为什么A会竭力隐藏自己的分配优势和各种数据。
但隐藏本身，就会产生另一种不信任。我们想象在实验中，B不知道总额是不是100元，只被告知自己可以被分配20元，那么B就会产生猜疑。
而A被分配的金额，最终会在“使用”环节被察觉，进而加深猜疑。其影响也不可小觑。比如明明总额是100元，却被猜疑成200元。
显然猜疑的成本有时候会更大。
最好的降低成本的方式，就是，透明。然后，倾向于B。B可以看到，然后就会从“消极”转为“积极”。
这玩意儿说白了确实很简单。也不需要经济学家反复对着每月数据念经。
然后给出各种荒了大唐的“建议”。
毕竟“天地之间有杆秤”。
